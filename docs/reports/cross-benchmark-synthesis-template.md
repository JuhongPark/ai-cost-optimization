# Cross-Benchmark Synthesis Template

## Purpose

This template defines the final write-up structure after the first empirical cycle completes.

## 1. Research Questions

Restate the three empirical questions:

1. Which baseline policy class is cheapest under a fixed quality threshold?
2. Which policy lever most changes the cost-quality frontier?
3. When is selective HITL cheaper than full automation or human-first handling?

## 2. Benchmark Winners

Summarize one line per benchmark:

- winner under threshold-constrained comparison
- winner under scenario-weighted comparison
- whether the two matched

## 3. Cross-Benchmark Patterns

Look for:

- task-specific optimal policies
- cases where indirect cost changed the winner
- cases where selective HITL dominated or failed

## 4. Policy-Lever Ranking

State which levers appeared highest value:

- stronger model
- routing
- retry
- verification
- tool use
- selective HITL

## 5. Sensitivity Summary

Report where rankings were stable and where they were assumption-sensitive.

Minimum dimensions:

- human review time
- downstream failure penalty
- quality threshold strictness

## 6. Practical Takeaways

Translate findings into practitioner-facing guidance:

- what to try first in coding
- what to try first in classification
- what to try first in extraction

## 7. Second-Cycle Priorities

State what deserves deeper follow-up:

- expanded datasets
- better evaluators
- more realistic human review models
- agent workflow extension
