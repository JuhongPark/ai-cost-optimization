# Experiments

This directory stores experiment designs, benchmark definitions, and policy comparison plans.

Suggested reading order: benchmark candidates → benchmark selection rationale → benchmark blueprint → policy templates → first empirical cycle plan → policy comparison matrix → execution checklist. The human-oversight study design is a focused sub-study that can be read independently.

## Benchmark Design

- [`benchmark-candidates.md`](benchmark-candidates.md): recommended benchmark candidates for the first research cycle.
- [`benchmark-selection-rationale.md`](benchmark-selection-rationale.md): rationale for why the current benchmark families fit the thesis.
- [`benchmark-blueprint.md`](benchmark-blueprint.md): initial benchmark design for AI cost optimization studies.

## First Empirical Cycle

- [`policy-templates.md`](policy-templates.md): reusable policy templates for early benchmark comparisons.
- [`first-empirical-cycle-plan.md`](first-empirical-cycle-plan.md): concrete questions, comparisons, and outputs for the first empirical cycle.
- [`policy-comparison-matrix.md`](policy-comparison-matrix.md): benchmark-by-benchmark policy matrix for the first empirical cycle.
- [`execution-checklist.md`](execution-checklist.md): run checklist for the first empirical cycle.

## Focused Sub-Study

- [`human-oversight-study-design.md`](human-oversight-study-design.md): study design for full automation vs selective HITL vs human-first.
